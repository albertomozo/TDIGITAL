## 🎯 Cómo usar

1. Leer `manual.md` para entender qué es Markdown y sus variantes (original, CommonMark, GFM).
2. Estudiar `sintaxis.md` para dominar reglas y ejemplos.
3. Resolver `ejercicios.md` y comparar con `soluciones.md`.
4. Consultar `futuro.md` para ideas de innovación con IA y flujos de trabajo.
5. El fichero de habilidades `guia_skill` para mejoras y escalar tu nivel de prompting

## 🧩 Extensiones y compatibilidad

- Usa GFM en GitHub/GitLab (tablas, checklists, tachado). 
- Para entornos estrictos, usa CommonMark.
- Aplica `markdownlint` y `remark-lint` para estilo y consistencia.

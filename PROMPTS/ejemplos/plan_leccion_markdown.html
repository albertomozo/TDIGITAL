Introducción a Markdown

Plan de lección · 20 minutos · Modalidad presencial o virtual

20 minutos totales
Principiantes
Máx. 20 estudiantes
Sin conocimientos previos
Objetivos de aprendizaje
OA1 Explicar qué es Markdown y para qué se usa en entornos digitales y laborales.
OA2 Aplicar la sintaxis básica: encabezados, listas y énfasis de texto.
OA3 Reconocer la utilidad de Markdown en herramientas cotidianas como GitHub, Notion o chats.
Desarrollo de la clase
🚀
0 – 5 min Introducción · ¿Por qué dar formato al texto?
Conectar con lo que los estudiantes ya conocen antes de mostrar código.
Pregunta de enganche: "¿Han usado negritas en WhatsApp o Slack? ¿Cómo lo hicieron?"
Mostrar el mismo párrafo: sin formato vs. con Markdown renderizado → impacto visual inmediato.
Definición sencilla: "Markdown es un lenguaje liviano de marcado que convierte texto plano en formato visual."
Mencionar dónde lo verán: GitHub, Notion, README, Discord, Jupyter, documentación técnica.
⚒️
5 – 15 min Desarrollo · Sintaxis en vivo
El/la docente escribe en tiempo real en un editor (StackEdit, Dillinger o VS Code) mientras los estudiantes replican en sus dispositivos.
Bloque 1 (3 min) — Encabezados: H1 con #, H2 con ##, H3 con ###. Mostrar jerarquía visual.
Bloque 2 (3 min) — Énfasis: **negrita**, *cursiva*, ~~tachado~~. Escribir una oración y aplicarlos.
Bloque 3 (4 min) — Listas: desordenadas con -, ordenadas con 1. 2. 3. Crear una lista de "mis apps favoritas".
# Mi primera nota en Markdown Esto es un párrafo con texto **en negrita** y texto *en cursiva*. ## Mis herramientas favoritas - Notion - GitHub - Slack ## Pasos para empezar 1. Abre un editor 2. Escribe en texto plano 3. Observa el resultado
🎯
15 – 20 min Cierre · Reto de 3 minutos + reflexión
Actividad de producción individual rápida para consolidar lo aprendido.
Reto: "Crea en Markdown tu presentación personal: nombre (H1), rol o carrera (H2), 3 habilidades en lista, una frase importante en negrita."
2–3 voluntarios comparten su resultado en pantalla o en el chat del aula virtual.
Cierre reflexivo: "¿En qué momento de tu semana usarías Markdown?" (1 respuesta por estudiante).
Compartir recurso de referencia rápida para que continúen practicando.
Materiales necesarios
💻
Dispositivo con navegador
PC, tablet o celular — sin instalación requerida
📝
Editor online
StackEdit.io, Dillinger.io o Markdown Live Preview
🖥️
Proyector o pantalla
Para mostrar el código y el resultado en tiempo real
📋
Cheatsheet Markdown
Tarjeta de referencia rápida (impresa o PDF compartible)
Evaluación formativa
📌 Durante la clase
  • Observación directa mientras replican la sintaxis
  • Detección de errores comunes: espacios faltantes, # mal ubicado
  • Preguntas rápidas de comprensión al grupo
✅ Reto de cierre
  • ¿Usó al menos 2 niveles de encabezado?
  • ¿Aplicó negrita o cursiva correctamente?
  • ¿La lista tiene formato válido (guion o número)?
💬 Reflexión final
  • "¿Qué fue lo más sorprendente de Markdown?"
  • "¿Dónde crees que lo usarás esta semana?"
  • Registrar respuestas para retroalimentación
📈 Extensión opcional
  • Crear un README básico para un proyecto personal
  • Explorar links e imágenes en Markdown
  • Subir ejercicio a GitHub como primer commit
